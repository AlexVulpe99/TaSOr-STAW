Developer Guide - TaSOr

Autori:
Alexandru Vulpe & Anghel Cristi

1. Introducere

Sa se realizeze un sistem online de tip PWA capabil sa gestioneze timpul de efectuare a unor activitati.

PWA(Progressive web application) presupune ca aplicatia noastra sa contina urmatoarele:

-sa funtioneze pentru orice user pe orice browser

-sa fie responsive

-sa aiba un grad de conectivity independence

-sa aiba o navigare tip aplicatie

-sa fie mereu la up-to-date

-sa fie safe

-sa nu fie greu de utilizat si inteles

2. Structura

Aplicatia la nivel de client va contine cod HTML, CSS si JAVASCRIPT.

Aplicatia la nivel de server va contine cod JAVASCRIPT si vom folosi baza de date relationala MySQL si NodeJS engine.

Tipologia preferabila pentru aceasta aplicatie este "Fat Server" - "Thin Client" cu serverul care va pune la dispozitie un REST API si clientul care va consuma API-ul.

3. Detalii de implementare

3.1. Backend

Baza de date MySQL care va avea principalele legaturi prezentate mai jos.

Codul din backend va contine clasele Task si User care vor face manipulare de informatii pe baza de date si vor avea urmatoarea structura:

Vom utiliza si module pentru Notificari, TaskResponse, TaskDescription si pentru statistici.

3.2 Frontend

Pentru a mentine cat mai simpla partea de client, vom folosi doar apeluri asincrone catre API.

3.3 API

Endpoint-urile vor fi:

'/register' - POST ==> creaza un nou user in baza de date : Clientul trimite informatiile structurate intr-un JSON.

'/login' - POST ==> se verifica parola : Clientul trimite informatiile structurate intr-un JSON

'/createdtasks' - GET ==> task-urile create de utilizator: Serverul trimite informatiile structurate intr-un JSON

'/asignedtasks' - GET ==> task-urile unui utilizator: Serverul trimite informatiile structurate intr-un JSON

'/taskresponse' - POST ==> raspunul unui task : task-urile create de utilizator: Clientul trimite informatiile structurate intr-un JSON

4. Flow-ul aplicatiei

Un user poate propune un task si poate adauga o persoana sa il execute intr-o perioada de timp specificata

Doar userul care a propus task-ul si cel care este adaugat la task poate viziona informatiile despre taskul respectiv

Userul care a propus task-ul poate expune public informatiile despre task

Fiecare task contine :

-interval orar sau data de finish

-descriere acestuia

-skill-uri/cunostinte necesare (optional)

-framework-uri necesare(optional)

Dupa finalizarea task-ului, userul asignat task-ului respectiv, adauga dovezi(un url, o poza, cod sursa, text, documente etc.)

Functionalitati:

-istoric al taskurilor : pentru fiecare user, pentru fiecare tip de task (task creat de el sau task la care a fost asignat) avem liste cu taskurile precedente

-statistici : pentru toti userii, pe o pagina separata vom oferi statistici (proprii si globale)

-notificari : acestea vor aparea in browser, pentru fiecare user
